## 発生した問題と解決の道のり

当初の機能実装は早期に完了していましたが、本番環境へのデプロイ（公開）の過程で、複数の複雑な問題が連鎖的に発生しました。原因の特定と解決に時間を要しましたが、一つ一つ着実に解決いたしました。以下にその詳細を記します。

| 問題 | 原因 | 解決策 |
| :--- | :--- | :--- |
| **1. 同期ボタンが表示されない** | Vercel（ウェブサイト公開サービス）が、開発に使用していたリポジトリとは別の古いリポジトリ (`dayservice-transport-planning`) を参照していました。 | 開発リポジトリの最新コードを、Vercelが参照しているリポジトリに強制的にプッシュし、内容を統一しました。 |
| **2. ビルドエラー (`vite: command not found`)** | Vercelのビルド設定で、プロジェクトのルートディレクトリが指定されておらず、ルートでビルドコマンドが実行されていました。 | Vercelの設定で「Root Directory」を正しく `transport-web` に指定しました。 |
| **3. ビルドエラー (`Could not resolve ...`)** | 印刷機能など、多数の関連ファイルがGitリポジトリに追加されておらず、Vercelがビルドに必要なファイルを見つけられませんでした。 | 未追跡になっていたすべての不足ファイルをGitリポジトリに追加し、プッシュしました。 |
| **4. Productionデプロイが実行されない** | VercelのProduction（本番）ブランチが `master` に設定されていましたが、私たちは `main` ブランチにコードをプッシュしていました。 | `master` ブランチを新たに作成し、最新のコードを `master` ブランチにプッシュすることで、Productionデプロイをトリガーしました。 |
